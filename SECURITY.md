# Security Policy

## Supported versions

Security fixes are applied to the latest published release on a best-effort basis.

## Reporting a vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Instead, email [info@sapkalabs.com](mailto:info@sapkalabs.com) with:

- a clear description of the issue,
- affected package version(s),
- impacted platform(s), and
- reproduction details or a proof of concept, if available.

We will review reports privately and coordinate a fix and disclosure process with the reporter.
